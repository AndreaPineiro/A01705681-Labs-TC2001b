Programación Orientada a Objetos

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

    2. Este proceso llamado validación de formulario en el lado del cliente ayuda a verificar que los datos se envíen de forma correcta y que coinciden con los requisitos de los controles del formulario.
    Es buena práctica pues garantiza una buena experiencia de usuario, facilitando la comunicación entre el servidor y el cliente. Además le ahorras trabajo al servidor, y es más rapido que esperar que te responda el servidor.

  2. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

    3. Se puede escribir simplemente novalidate en el inspector de la página para saltarse una validación. Esto desactiva la validación automática del navegador.
    La validación del lado del cliente es muy fácil de evitar, por lo que es muy fácil para los usuarios pueden enviar datos incorrectos al servidor. Por lo que también debe hacerse una validación en el lado del servidor.

  3. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

    4. Como se mencionó mejora la experiencia de usuario, pues detectan datos inválidos para que el cliente los corrija inmediatamente.
    Se usa principalmente para verificar que el usuario no cometió algún error de manera accidental; de esta manera el usuario recibe retroalimentación y le permite tener una buena experiencia en la página.
